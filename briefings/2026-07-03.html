Since Linux 6.9, LUKS Suspend Stopped Wiping Disk-Encryption Keys From Memory

483 pts · 208 comments · → Read more · Discuss on HN

A regression since kernel 6.9 means the LUKS suspend-to-RAM path no longer reliably wipes disk-encryption keys from memory before sleep, so a seized or stolen laptop's keys can persist in RAM and be pulled via a cold-boot or DMA attack. The bug affects any distro relying on systemd's default suspend hooks and has no CVE yet, though the discoverer has flagged it upstream.
